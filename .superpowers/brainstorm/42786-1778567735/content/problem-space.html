Aegis AI 评估层 — 第一个问题

我们聊了很多，先对齐：我们要解决的核心痛点是哪个？

717GB 事故复盘

用户说：
"删除桌面 WIP 文件夹"
AI 生成：
cmd /c "rd /S /Q \"C:\Users\ADMIN\Desktop\WIP\""
经过 zsh → tmux → PowerShell → cmd 后变成：
rd /S /Q \　　← C盘根目录，717GB 没了
Aegis 看到的命令：
cmd /c "rd /S /Q \"C:\Users\ADMIN\Desktop\WIP\""　← 看起来合法，不会拦截

这个故事里真正的痛是什么？选一个最准的：

A

命令变质了，用户没察觉

AI 生成的命令经过多层解析后面目全非，但执行时没有任何提示。用户以为在删文件夹，其实在删 C 盘。

→ 解法：执行前展示命令的真实效果

B

高危命令没有人工确认环节

删 313GB 这种量级的操作，不应该 AI 说执行就执行。应该强制人工确认，让人看一眼再决定。

→ 解法：高危操作必须人工审批

C

用户看不懂命令，无法判断风险

用户看到 rd /S /Q 根本不知道是干嘛的。就算弹了确认框，看不懂也还是会点"允许"。

→ 解法：AI 把命令翻译成人话

D

AI 的意图和命令不一致，没人发现

用户说"删文件夹"，AI 生成了"删 C 盘"的命令。这个不一致应该被检测出来并拦截。

→ 解法：AI 比对用户意图 vs 实际命令
