Aegis 新功能：三个方向

基于 7 篇真实案例提炼的痛点 — 选一个或告诉我你的想法

A

Impact Preview（影响预览）

拦截破坏性命令后，先跑一遍只读侦查（find/ls/wc），在审批窗里展示真实影响：

⚠ 命令将删除以下内容：
路径: /Users/yuhao/project/cache/
文件数: 3,847 个文件
总大小: 12.3 GB
⚠ 注意: 路径在当前项目目录之外

优势

  • 不需要 AI API，纯本地
  • 零延迟，技术简单
  • 直接解决 6/7 个案例

局限

  • 只能预览，不能判断意图
  • SSH/数据库远程操作拿不到真实数量
B

AI Risk Explainer（AI 风险解释器）

拦截命令后，调用 AI（用户自带 API Key）把命令翻译成人话 + 给出风险判断：

Aegis AI 解读：
这个命令会永久删除 /Users/yuhao/Downloads/ 目录下的 old-project-credentials.json 文件。
⚠ 风险：该文件路径不属于当前项目（当前项目: /Users/yuhao/myapp），可能是误操作。
💡 建议：如需删除凭证文件，请先确认这是否为目标项目的文件。

优势

  • 能解释复杂命令
  • 能检测路径在项目外
  • 给安全替代建议

局限

  • 需要 API Key，破坏纯本地
  • 每次拦截多 1-2 秒延迟
  • 有 API 成本
C

A + B 组合：本地预览 + 可选 AI 解释

Impact Preview 作为默认开启的本地功能，AI 解释作为可选增强层（填 API Key 才开启）：

⚠ 影响预览（本地）：
将删除 3,847 个文件 / 12.3 GB
路径在当前项目外 ⚠
──── AI 解释（需 API Key）────
这个命令误操作概率高，建议拒绝执行。

优势

  • 默认体验不依赖网络
  • 有 API Key 的用户体验更好
  • 覆盖所有案例

局限

  • 实现工作量最大
  • 需要设计好两层的 UI
