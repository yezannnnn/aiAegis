自动拒绝

快速开始

三步安装，即刻保护你的 AI Agent

1

安装 Aegis

全局安装 CLI 工具

# 安装 Aegis
npm install -g aegis
2

配置规则集

启用你需要的技术栈规则

# ~/.aegis/rules.yaml
extends:
  - aegis/recommended
  - aegis/mysql
  - aegis/nestjs
  - aegis/prisma
  - aegis/vue3
  - aegis/git
3

启动 Monitor

启动 TUI 监控界面

# 启动 Monitor
aegis monitor

# 或在项目中运行 Agent
aegis run npm run dev

常见问题

关于 Aegis 的疑问解答

Aegis 会拦截所有命令吗？ +

不会。Aegis 采用分层策略：只有匹配规则的高危操作才会被拦截。日常开发命令（如 ls、cat、npm install）默认放行。你可以通过规则集配置调整拦截粒度。

如何添加自定义规则？ +

在 ~/.aegis/rules.yaml 中添加 extends 引用自定义规则文件，或直接编写规则覆盖。规则格式为 YAML，支持正则匹配和 severity 设置（block/error/warn/off）。

Monitor TUI 支持远程审批吗？ +

当前版本通过本地 Unix Socket 通信，Monitor 和 Agent 需在同一个机器上。远程审批功能在 Roadmap 中，计划通过 WebSocket 或 HTTP API 实现。

可以集成到 CI/CD 吗？ +

可以。Aegis 提供非交互模式（--no-monitor），此时高危操作直接拒绝并返回错误码，适合 CI/CD 流水线使用。也可以通过环境变量 AEGIS_ENABLED=0 临时禁用。

支持 Windows 吗？ +

支持。Aegis 使用 TCP Socket 通信，跨平台兼容 macOS、Linux 和 Windows。安装方式相同：npm install -g aegis
