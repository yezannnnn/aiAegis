# Aegis Socket架构 - 用户初始化流程分析

## 🔧 当前Socket方案的用户设置步骤

### 第一阶段：环境准备
1. **安装Node.js** (如果用户没有)
   - Windows: 下载安装包
   - Mac: brew install node 或下载安装包
   - ⚠️ 复杂度：中等 (需要额外软件)

2. **安装依赖包**
   ```bash
   npm install ws
   ```
   - ⚠️ 复杂度：简单 (但需要命令行操作)

### 第二阶段：文件部署
3. **下载Aegis文件**
   - aegis-backend-service.js
   - hook-with-backend.js
   - websocket-client.html
   - ⚠️ 复杂度：简单 (但文件较多)

4. **配置Hook路径**
   ```bash
   # 创建.aegis目录
   mkdir ~/.aegis
   # 复制Hook文件
   cp hook-with-backend.js ~/.aegis/claude-hook.js
   ```
   - ⚠️ 复杂度：中等 (需要命令行操作)

### 第三阶段：服务启动
5. **启动后端服务**
   ```bash
   node aegis-backend-service.js
   ```
   - ⚠️ 复杂度：中等 (需要保持终端运行)

6. **配置Claude Code Hook**
   ```bash
   export CLAUDE_PRETOOLUSE_HOOK="node ~/.aegis/claude-hook.js"
   ```
   - ⚠️ 复杂度：高 (环境变量配置)

7. **打开Web监控界面**
   - 浏览器打开 websocket-client.html
   - ⚠️ 复杂度：简单

### 第四阶段：验证测试
8. **测试系统**
   - 执行危险命令验证拦截
   - 检查Web界面显示
   - ⚠️ 复杂度：简单

## 📊 复杂度评估

### 用户技能要求
- ✅ 基础：浏览器操作
- ⚠️ 中级：命令行操作
- ⚠️ 高级：环境变量配置
- ⚠️ 高级：多进程管理

### 步骤数量统计
- 总步骤数：8个
- 命令行操作：5个
- 文件操作：3个
- 配置操作：2个

### 失败风险点
1. Node.js安装失败
2. 依赖包安装失败
3. Hook路径配置错误
4. 环境变量配置错误
5. 端口冲突 (9876, 9877)
6. 后端服务意外退出

## 🎯 复杂度评级：⚠️ 较复杂 (7/10)

### 与其他方案对比

| 方案 | 步骤数 | 命令行 | 技能要求 | 复杂度 |
|------|--------|--------|----------|--------|
| **Socket架构** | 8 | 5个 | 中-高级 | ⚠️ 7/10 |
| **文件队列** | 4 | 2个 | 初级 | 🟢 3/10 |

## 💡 用户痛点分析

### 主要困难
1. **多组件管理**：需要同时管理后端服务、Hook、Web界面
2. **进程持久化**：后端服务需要持续运行
3. **端口管理**：需要确保9876、9877端口可用
4. **故障排查**：多个组件，故障点增多

### 用户抱怨预测
- "为什么需要安装这么多东西？"
- "后端服务总是停止，很麻烦"
- "端口冲突怎么解决？"
- "命令行操作太复杂了"
